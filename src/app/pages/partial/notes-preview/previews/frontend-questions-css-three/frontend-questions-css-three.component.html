21. What is the CSS display property, and can you give a few examples of it's use?

The display property specifides how an element is displayed (the type of rendering box). Few examples:

  • none, block, inline, inline-block.

  • table, table-row, table-cell, list-item (these let the element behave the like the respective html tags these refer to: <table>, <tr>, <td>, <li>)

22. What's the difference between inline and inline-block?

block:

  • element's width = parent's width

  • element starts on a new line, and does not let other elements to sit inline with it.

  • cannot use the vertical-align property on it.

block and inline-block similarities:

  • can specify width and height of element.

  • all side of margin and padding are respected, if applied.

inline-block and inline similarities:

  • element's size depends on it's content

  • element allows other elements to sit inline with it.

  • can use the vertical-align property on it.

inline:

  • ignores width and height properties, if set.

  • only horizontal sides of margin and padding are respected, if applied.

  • when applying float, it acts like a block element (such that block properties apply.)

What's the difference between a relative, fixed, absolute, and static-ally positioned element?

A positioned element: has a position property that is set to either relative, fixed, absolute or sticky (aka anything but static).

The possible values for a position property:

  • static: this is the default setting; the element is in the flow of the page. The top, right, bottom, left and z-index properties are ignored if set.

  • relative: element's position is adjusted from where it was originally (if it was static). Layout is not changed from this adjustment (flow is the same).

  • absolute: element is positioned as specified (by top, left, bottom, and/or left properties) based on its closest ancestor that is also positioned (not static). If this element dosen't exist, it is positioned relative to its parent. An absolute element can have a margin, and don't affect the positioning of other elements (read: they are removed from the flow of the page).

  • fixed: element is positioned as specified, relative to the viewport (visible screen). A fixed element does not move as the window scrolls. It is also removed from the flow of the page.

  • sticky: element is relatively positioned -- until it passes a specified scroll height. After that it is fixed positioned.

What existing CSS frameworks have you used locally, or in production? Any comments on them?

A couple of popular CSS frameworks:

  • Bootstrap: easy to learn and customize for your own theme.

  • Semantic UI: difficult to learn, and difficult to customize.

Have you played around with the new CSS Flexbox or Grid specs?

Flexbox and CSS Grid is used to layout content. Flexbox solves common CSS issues, such as vertical centering of elements in a container. It is used by Bootstrap. Using the Grid system is more intuitive, but it doesn't have as much browser support as Flexbox yet.
