21. What is the CSS display property, and can you give a few examples of it's use?

The display property specifides how an element is displayed (the type of rendering box). Few examples:

  • none, block, inline, inline-block.

  • table, table-row, table-cell, list-item (these let the element behave the like the respective html tags these refer to: <table>, <tr>, <td>, <li>)

22. What's the difference between inline and inline-block?

block:

  • element's width = parent's width

  • element starts on a new line, and does not let other elements to sit inline with it.

  • cannot use the vertical-align property on it.

block and inline-block similarities:

  • can specify width and height of element.

  • all side of margin and padding are respected, if applied.

inline-block and inline similarities:

  • element's size depends on it's content

  • element allows other elements to sit inline with it.

  • can use the vertical-align property on it.

inline:

  • ignores width and height properties, if set.

  • only horizontal sides of margin and padding are respected, if applied.

  • when applying float, it acts like a block element (such that block properties apply.)

23. What's the difference between a relative, fixed, absolute, and static-ally positioned element?

A positioned element: has a position property that is set to either relative, fixed, absolute or sticky (aka anything but static).

The possible values for a position property:

  • static: this is the default setting; the element is in the flow of the page. The top, right, bottom, left and z-index properties are ignored if set.

  • relative: element's position is adjusted from where it was originally (if it was static). Layout is not changed from this adjustment (flow is the same).

  • absolute: element is positioned as specified (by top, left, bottom, and/or left properties) based on its closest ancestor that is also positioned (not static). If this element dosen't exist, it is positioned relative to its parent. An absolute element can have a margin, and don't affect the positioning of other elements (read: they are removed from the flow of the page).

  • fixed: element is positioned as specified, relative to the viewport (visible screen). A fixed element does not move as the window scrolls. It is also removed from the flow of the page.

  • sticky: element is relatively positioned -- until it passes a specified scroll height. After that it is fixed positioned.

25. Have you played around with the new CSS Flexbox or Grid specs?

Flexbox and CSS Grid is used to layout content. Flexbox solves common CSS issues, such as vertical centering of elements in a container. It is used by Bootstrap. Using the Grid system is more intuitive, but it doesn't have as much browser support as Flexbox yet.

26. Can you explain the difference between coding a website to be responsive, versus using a mobile-first strategy?

These two approaches are similar, but with one key difference. Responsive design means elements are designed such that they adapt to different device widths and heights (typically only width). This is done through @media queries, for example.

A mobile-first strategy follows responsive design, but defines its default styles for mobile displays first. Then, elements can be adapted for non-mobile devices.

Advantages of mobile-first:

  • Better performance for mobile: CSS rules are applied by default, and don't need to be checked by @media queries.

  • Cleaner code.

Responsive:

@media (min-width: 601px) {
  .my-class {
    font-size: 24px;
  }
}
@media (max-width: 600px) {
  .my-class {
    font-size: 12px;
  }
}

Mobile-first:

.my-class {
  font-size: 12px;
}
@media (min-width: 601px) {
  .my-class {
    font-size: 24px;
  }
}

27. How is responsive design different from adaptive design?

Similarities: both approaches try to optimize user experience for different devices (viewport size, resolution, inputs).

Responsive design tries to make its web-app flexible:

  • One website design that is made to adapt to different devices.

  • Website adjusts its layout; not exactly delivering an entirely new layout for each device.

  • Using @media quesries, grid systems, responsive images, etc.

Adaptive design creates a new layout for different devices.

  • Website detects device being used, then delivers a pre-defined layout that it has created for that type of device.

  • Instead of adjusting one layout, deliverys an entirely new layout for each device.

28. Have you ever worked with retina graphics? If so, when and what techniques did you use?

Retina screens have resolutions where the pixel ratio is greater than 1. Note that browsers render DOM elements based on the device's resolution, with the exception of images.

In order to optimize images for retina displays, we need to use high-resolution images (but only for these retina displays!). For that, we can use srcset to display responsive images, as discussed previously.

29. Is there any reason you'd want to use translate() instead of absolute positioning, or vice-versa? And why?

translate() is used for the CSS transform property. It is good becuse it does not trigger reflow or repaint. It only triggers composition. On the other hand, absolute positioning triggers reflow... which uses CPU resources, and it thus inefficient.

The other difference is that when using translate(), the element remains in the flow of the webpage (read: takes up space). As discussed, absolute positioning takes elements out of the flow of the page.
