13. Can you give an example of an @media property other than screen?

There are four types:

  • all: all media type devices.

  • print: printers.

  • speech: screen-readers that "read" a page out loud.

  • screen: device screens.

When using these @media properties, you specify a style just for those specific media outputs.

14. What are some of "gotchas" for writing efficient CSS?

Browsers match selectors from the right-most selector, then proceeds leftwards. In a selector such as .card .title span, the browser tries to match span first, and then filters all span elements by .title, and so on. This is not good because span matches a lot of elements! We try to avoid having key selectors that are tag or universal.

We must also be aware which CSS properties trigger reflow, repaint, and compositing. We want to only apply proprties that trigger compositing (and avoid ones that trigger repaint and reflow, because those require the browser to do more work).

15. What are the advantages and disadvantages of using CSS preprocessors ?

Advantages:

  • Maintainable and legible CSS.

  • Easier to write nested selectors.

  • Variables for theming.

  • Mixins for repeated/patterned CSS.

  • Modular: CSS codebase can be split into different files and then imported.

Disadvantages:

  • Need tools for preprocessing to write production ready CSS.

16. How would you implement a web design comp that uses non-standard fonts?

We can either:

  • Use @font-face to get the desired font from a url (or locally).

  • Import the font from a url through @import.

And then we define the font-family, and font-weight for the element(s) that require different font styling.

17. Explain how a browser determines what elements match a CSS selector.

Explained already above: the browser matches a CSS selector from the right-most selector on the chain, the finds all the DOM elements matching it. Then from those matching elements, the browser filters them out by the next selector (left) , and so on, until it has matched all appropriate elements (or browser has filtered down to one or no elements).

Efficiency tips:

  • Write shorter CSS selector chains.

  • Ensure that the right-most (key) selector in a chain doesn't match a lot of elements (like universal or tag selectors).

18. Describe pseudo-elements and discuss what they are used for.

CSS pseudo-elements are selectors that allow to style a specific part of an element. They allow for styling/content changes without modifying HTML.

Examples:

  • :first-line and first-letter for decorating text.

  • .clearfix for adding a zero-space element (without changing markup!).

  • :before and :after (combined with content) for adding elements that appear to be in the DOM (without changing markup!) 

h1::after {
  content: "hello"
}

Appears like:

<h1>
  Other h1 content
  hello
</h1>

When using ::before, then the content will appear before the rest of the content in the element. 

h1::before {
  content: "hello"
}

appears like:

<h1>
  hello
  Other h1 content
</h1>

19. Explain your understanding of the box model and how you would tell the browser in CSS to render your layout in different box models.

HTML elements are represented by rectangular boxes. Each box contains:

  • Content area: where text, images, and divs are contained.

  • Padding area (optional): an area surrounding content, separating them from the border.

  • Border area (optional): an area consisting of the border, which surrounds the padding area.

  • Margin area (optional): an area surrounding the border, separating the box from other content.

The box model is responsible for some calculations about these boxes. The calculations:

  • Amount of space taken up by the block element.

  • Whether or not the borders/margins overlap (the sum of two adjacent margins are combined) or collapse (two adjacent margins collapse to one margin).

  • Dimensions of the box.

Box model rules for block elements:

  • Dimensions are calculated using the width, height, padding, border and margin.

  • When no height is specified: element will take height of its content, plus padding (exception: floats).

  • When no width is specified: element will take width of its parent, minus padding (exception: floats).

  • Height of element = height of its content (default with box-model: content-box).

  • Width of element = width of its content (default with box-model: content-box).

  • By default, border and padding are not used in calculating an element's width and height.
